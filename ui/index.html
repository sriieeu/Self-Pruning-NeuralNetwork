Theory

How Hard Concrete Gates Work

The mathematics behind differentiable neuron-level pruning via the BinaryConcrete distribution.

01

Log-Alpha Parameter

Each output neuron gets a learnable scalar log α. At initialisation, log α = 0, giving each neuron a 50% probability of being active — a neutral prior.

self.log_alpha = nn.Parameter(torch.zeros(n_neurons))
02

Stochastic Gate (Training)

During training, a soft gate is sampled via the reparameterisation trick — gradients flow through log α cleanly:

u ~ Uniform(0,1)
s = σ((log u − log(1−u) + log α) / β)
s̄ = s × (ζ − γ) + γ   # stretch
gate = clamp(s̄, 0, 1)   # hard clamp
03

Deterministic Gate (Inference)

At test time, the stochasticity is removed. The gate is a hard binary decision — no randomness, fully sparse sub-network:

gate = 1[σ(log α) > 0.5]
04

L0 Penalty

The penalty is the expected fraction of active gates. Minimising it drives log α negative, making neurons increasingly likely to be pruned:

L0 = Σᵢ σ(log αᵢ − β × log(−γ/ζ))
05

Stretch Parameters

ζ = 1.1 and γ = −0.1 are the stretch bounds from the paper. They push probability mass to exactly 0 and 1, giving truly sparse solutions — not just near-zero weights.

ζ = 1.1   # upper bound
γ = −0.1  # lower bound
# P(gate = 0) > 0 and P(gate = 1) > 0
06

Why It Works

L1 on weights encourages small weights but never forces exact zeros. L0 counts non-zeros directly. The Hard Concrete makes L0 differentiable — best of both worlds.

# L1 pushes weights small (not zero)
# L0 pushes neurons dead (exactly zero)
# Hard Concrete: differentiable L0 ✓
